# bench/attr-corpus/

A hermetic corpus of literal attribute values extracted from real
open-source Terraform code. Drives the microbenchmarks for the
format-validation check family (CIDR, ARN, region, account-ID, …).

## Why a corpus

Grammar-family checks fire on the string value of specific attributes
(`cidr_block`, `role_arn`, `region`, `account_id`, …). Synthetic
benchmarks with hand-crafted inputs miss two things:

1. **Realistic value distribution.** IPv4 CIDRs from real modules cluster
   around `/16` and `/24` prefixes; a hand-picked input set will not.
2. **Realistic attribute-name coverage.** Real repos use hundreds of
   distinct `*_arn` attribute names; our check trigger list is a curated
   subset of that population.

Committing the extracted values guarantees byte-identical benchmark input
across contributor machines and CI runs. Refreshing the corpus is an
explicit, reviewable diff.

## Layout

```
bench/attr-corpus/
├── README.md              # this file
├── repos.txt              # pinned <owner>/<repo> <tag> lines
├── fetch.sh               # downloads pinned tarballs into files/
├── extract.sh             # runs the Go extractor against files/
├── cmd/extract/main.go    # hclsyntax-based extractor (build-tag ignored)
├── files/                 # (gitignored) fetched .tf sources
└── values/                # committed extracted values, one per line
    ├── cidr.txt
    ├── arn.txt
    ├── region.txt
    └── account_id.txt
```

`files/` is generated by `fetch.sh` and gitignored (~200 MB when fully
fetched). `values/` is committed (a few KB total) so every checkout has
the corpus available without a network fetch.

## Regenerating the corpus

```sh
make bench-corpus-fetch     # download tar.gz snapshots into files/
make bench-corpus-extract   # refresh values/ from files/
make bench-corpus-clean     # remove files/ (keeps values/)
```

Or in one step:

```sh
make bench-corpus-refresh   # fetch + extract
```

After refreshing, review the diff to `values/` and commit if the numbers
look sensible.

## Bumping pinned tags

1. Edit `repos.txt` — bump the tag on any line, or add/remove repos.
2. Run `make bench-corpus-clean bench-corpus-refresh`.
3. Review `git diff bench/attr-corpus/values/`. Typical change: a handful
   of new CIDRs/ARNs, a few obsolete ones dropped.
4. Commit the `values/` diff separately from any check-code changes so
   corpus bumps are easy to bisect against benchmark regressions.

## Extractor semantics

`cmd/extract/main.go` parses every `.tf` file with hclsyntax, walks all
attributes (including those inside nested blocks) and matches attribute
names against per-family regex patterns:

| Family     | Pattern                                              |
|------------|------------------------------------------------------|
| cidr       | `(^\|_)cidr(_blocks?)?s?$`                           |
| arn        | `(^\|_)arns?$`                                       |
| region     | `^(aws_)?region$`                                    |
| account_id | `^(allowed_\|aws_\|master_\|caller_)?account_ids?$`  |

For each match the extractor pulls the literal string value (or list of
strings), skipping any expression that contains interpolation (`${…}` or
`%{…}`). Skipped inputs are exactly the ones our checks decline to
validate — the corpus mirrors that discipline.

Extractor patterns are deliberately **wider** than the trigger lists the
checks ship: we want representative benchmark input, not undersampling.
The checks themselves keep tight, curated allow-lists per the design
principles in issue #23.
